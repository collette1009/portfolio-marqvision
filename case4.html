CRM 필드를 고객 의도 기반으로 재구성해 재확인 왕복을 95%+ 제거했습니다

환불·교환 CRM 필드가 자유 입력 구조라 CS팀이 고객에게 다시 확인해야 하는 왕복이 잦았습니다. 고객 의도 기반 선택값으로 재구성해 재확인 왕복을 95%+ 제거하고, 입력 구조가 후속 처리 흐름과 자동으로 연결되도록 만들었습니다.

문제 상황

자유 입력 필드가 처리 흐름과 끊겨 있었습니다

환불·교환 CRM 필드가 자유 입력 방식이라 같은 의도(반품·교환·교환 후 재반품 등)도 다르게 적혔고, CS팀이 매번 고객에게 다시 확인해야 했습니다. 입력 단계와 후속 처리(정산·재고·물류)가 연결되지 않아 재확인 왕복이 운영 리드타임을 늘리고 있었습니다.

What I Did

입력 구조를 처리 흐름과 연결되는 형태로 재설계했습니다

실행 A · 고객 의도 분류 기준 정의

환불·교환 케이스를 사례 분석해 고객 의도를 명확한 axis로 정리했습니다. 자유 입력에서 의도 분류로 옮길 수 있는 기반을 만들었습니다.

실행 B · 선택값 기반 필드 재구성

환불·교환 CRM 필드를 자유 입력에서 고객 의도 기반 선택값으로 재구성했습니다. 개발 범위 최소화 스펙으로 협의해 빠르게 적용했습니다.

실행 C · 후속 처리 흐름과 연결

선택값이 정산·재고·물류 후속 처리에 자동 반영되도록 흐름을 정렬했습니다. CS 재확인 왕복이 95%+ 사라지고 처리 리드타임이 단축됐습니다.
