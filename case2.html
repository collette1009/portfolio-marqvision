Situation

반복 입력이 분류 오류와 처리 지연을 만들고 있었습니다

재배송·반품 처리가 8단계 반복 입력으로 운영됐습니다
같은 정보(주문번호·귀책·사유·금액)를 8단계에 걸쳐 수동 입력해야 했습니다. 입력 누락·오타·중복이 분류 오류로 이어지고, 처리 리드타임을 늘렸습니다.
귀책 기준이 사람마다 달라 분류 편차가 컸습니다
고객 귀책·물류사 귀책·자사 귀책을 판단하는 기준이 명문화되지 않아 담당자별로 다르게 분류됐습니다. 분류가 다르면 정산도 다르게 흘렀습니다.
비개발 직군에서 AI 도구를 실무 도입한 사례가 없었습니다
회사는 Google Workspace 기반 Gemini Pro 사용 환경을 제공하고 있었지만, 운영 업무에 ChatGPT·Claude를 들여올 공식 경로는 없었습니다.

What I Did

Claude로 직접 만들어, 입력 단계와 분류 오류를 구조적으로 끊었습니다

Step 01
기존 8단계 프로세스 매핑 — 반복 입력·귀책 분기 식별
처리 흐름 전 구간을 단계별로 매핑해 반복 입력 구간과 귀책 판단 분기 지점을 찾았습니다. 자동화할 수 있는 곳과 사람 판단이 필요한 곳을 분리했습니다.
Step 02
Claude로 Chrome Extension 직접 구현 — 입력 단계 자동 축약
비개발 직군이지만 Claude로 코드 없이 Chrome Extension을 직접 구현했습니다. 주문번호 1회 입력으로 후속 7단계가 자동 채워지는 구조로 8단계를 2단계로 축소했습니다.
Step 03
귀책 기준값 자동 반영 로직 설계 — 분류 오류 구조적 차단
귀책 판단 기준을 코드 안에 명문화하고 자동 반영했습니다. 사람이 매번 판단하지 않아도 같은 조건에서 같은 결과가 나오도록 — 분류 편차를 구조적으로 제거했습니다.

AI Tooling Context

AI는 도구이지 자랑이 아닙니다 — 실무 효용이 인정받았습니다

Google Workspace 기반 Gemini Pro 사용 환경이 있었음에도, 업무 특성상 ChatGPT와 Claude를 병행 활용했습니다. 반복 운영 문서화, 정책/약관 검토, 리스크 문구 정리, 자동화 설계에서 실제 효용이 확인되어 — 사내 최초이자 비개발 직군 중 유일하게 유료 AI 도구 사용 비용을 지원받았습니다. AI를 답변 자동 생성에 쓰는 게 아니라 운영 품질을 높이는 도구로 적용했고, 회사가 그 효용을 인정한 결과입니다.
